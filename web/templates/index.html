常见问题

什么是 API 中转站鉴定?

这个工具用于快速判断一个 Claude API 中转站到底是稳定转发真实模型接口,还是存在包装、伪装、协议不一致或能力掺水的情况。Veridrop 会通过多轮结构化探测请求,对接口返回结构、知识表现、身份一致性、思维链痕迹和加密签名指纹进行交叉判断,得出 0–100 的综合评分。

它不是法律意义上的审计报告,也不承诺百分之百准确,但可以在你购买、集成、长期使用某个中转站之前,做一次低成本的技术核验,显著降低踩坑风险。

什么是 API 中转站?

API 中转站(也叫"镜像站"、"代理站"、"分发站")通常位于用户和上游模型服务(如 Anthropic 官方 API)之间,负责转发请求、统一计费、切换渠道,或者把多个模型接口包装成同一种调用方式。

它的价值在于:接入方便(无需海外信用卡和身份验证)、价格灵活(可批发议价)、可做聚合与分发。但同时也意味着你并不是直接面对原始模型提供商,中间多了一层不可控的环节。

API 中转站有什么样的隐患?常见骗局有哪些?

Claude API 中转站市场鱼龙混杂,常见隐患包括:

  • 模型身份被替换:你以为充的是 Claude Opus,实际转给了 Sonnet、Haiku,甚至换成了别家模型(如 GPT、Kiro IDE 等)。
  • 能力降级:PDF、Tool Use、Thinking 等高级功能被中转站静默剥离,你的请求看似成功,实则没用上完整能力。
  • 返回协议与官方不一致:消息 ID 用 UUID 替代 msg_,tool_use ID 硬编码为 tool_1,SSE 事件序列残缺。
  • 知识表现异常:模型回答 Anthropic 公司常识题时频繁错误,自报身份是其他厂家。
  • 响应被二次加工:输入 token 数与官方不一致(被注入隐藏 prefix)、输出被截断或过滤。
  • 密钥与日志透明度低:你的 API key 被记录、用于其他用户、甚至转售,无法核实。
  • 服务稳定性差:依赖单一上游账号,一旦封号、限流,所有用户一起断流。
  • 跑路风险:充值大额后短时间内站长卷款消失。

Veridrop 的 10 项检测正是针对以上每一类隐患设计——比如 Identity Consistency 检测身份替换,Structured Output 检测 tool_use 假 ID,Response Integrity 检测输入注入。

报告里的 TTFT、LATENCY、TOKENS/S 是什么意思?
  • TTFT (Time to First Token):首字节(实际是首 token)时间,从发出请求到收到第一个 content_block_delta 事件的耗时。Veridrop 取所有流式请求中的最小值——代表中转站在最简单的提示下的响应速度,排除模型思考开销,反映的是纯网络/调度延迟。低于 1 秒优秀,2 秒以上意味着上游可能在排队或路由绕远。
  • LATENCY:整批检测的钟表时间,从 Runner 启动到所有 detector 完成。由于检测是并行执行,这个数 ≈ 最慢那个 detector 的耗时,而不是 13 个请求耗时之和。完整模式正常落在 20–40 秒。
  • TOKENS/S:输出 tokens 数 ÷ LATENCY,粗略吞吐参考。
  • INPUT TOKENS / OUTPUT TOKENS:本次检测累计的输入与输出 token 数,可与官方 baseline 对照——若 INPUT 显著偏高,说明中转站可能在请求里偷偷注入了 system prompt。

检测分数高的中转站,充值就绝对安全吗?

不是。 技术指标不等于信誉保证。Veridrop 的算法能识别中转站接口当前是否真诚、稳定,但无法预测站长的经营持久力。

第三方中转站受上游政策影响较大,行业内服务商变动频繁——上游突然封号、调价、停供,都会让原本检测全过的站点立即不可用。为了你的资金安全,请始终保持风险意识:

  • 初次使用仅做最低限度充值(20–50 元)
  • 使用 1–2 周后再小额续费,避免一次性大额囤积
  • 定期(每周或每月)用 Veridrop 重测同一家中转站,看分数是否稳定
  • 关注社群、TG 群、推特等渠道的实时反馈,出现跑路苗头立刻减少充值

记住:没有任何检测工具能替代风险分散。即便是 100/100 的中转站,也建议同时保留 1–2 个备选,避免单点依赖。

我的 API 密钥会不会被泄露或滥用?

不会。Veridrop 的密钥处理流程:

  • 提交时通过 HTTPS 加密传输到 Veridrop 服务器
  • 仅在本次检测的内存中保留,用完立即丢弃
  • 持久化的 JSON 报告中只保存掩码形式(如 sk-ab••••••cd),无法还原原始密钥
  • 密钥不写入日志、不进数据库、不上传第三方
  • 检测过程中只对你提供的 base_url 发起请求,不会被用于其他用途

但出于风险意识,仍建议:使用专门用于测试的 API key(余额少、可随时吊销),而不是你日常生产环境的主 key。

能不能检测 OpenAI / GPT / Gemini 中转站?

暂不支持。Veridrop 当前只针对 Anthropic Messages API 协议(POST /v1/messages)。OpenAI Responses/Chat Completions 协议、Google Gemini 协议都在规划中,但因为这两家没有像 Claude 的 thinking signature 那样的加密级反伪指纹,鉴定难度更大,需要换一套检测维度。

如果你的中转站只走 Anthropic 协议(包括 Claude Opus 4.7、Sonnet 4.6、Haiku 4.5 等),Veridrop 完全适用。

检测失败 / 报错 / 超时怎么办?

常见原因和处理:

  • "No available channel for model XXX":中转站没开通该模型的路由。换一个模型试试,或联系站长开通。
  • HTTP 401 / 403:API key 无效或没权限。检查 key 是否输入正确、是否过期。
  • HTTP 429:被中转站限流。等几分钟再试,或换个时间段。
  • 超时:中转站延迟过高(>180 秒),通常是上游过载。可以先用快速模式 (quick) 测一下能不能通,再尝试完整模式。
  • 网络错误:base_url 拼写错了,或站点已下线。先用 curl 试试 {base_url}/v1/messages 是否可达。
