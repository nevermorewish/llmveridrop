05

完全开源 · 不存「偷 key」可能

API key 只在内存里跑完即丢,不写日志、不写磁盘。代码全开源,你可以审计服务端,或者直接 clone 部署到自己机器上 — 比闭源「黑盒检测」工具透明得多。

Veridrop 不只给一个「75 分」总分 — 每项扣分都附具体证据:usage 字段缺了哪些键、stream 漏了哪个事件、tool_use ID 是不是规范前缀、签名长度异常多少。一眼看出中转站在哪个环节动了手脚,不是黑盒一个数字。

怎么用?三步 60 秒

  1. 1

    填中转站地址 + API key

    选要测的协议页(Claude / OpenAI / Gemini),粘贴 base_url 和 key,模型下拉自动列出该 key 在该中转站可用的模型。

  2. 2

    点「开始检测」,等 30-70 秒

    Veridrop 并发跑 7-10 项检测,实时显示进度。期间消耗的 token 由你的 key 支付(Haiku 约 $0.012,GPT-4o-mini 约 $0.005)。

  3. 3

    看报告,重点看红黄项

    总分 ≥85 优秀(绿) / 70-84 通过(浅绿) / 50-69 存在风险(黄) / <50 未达标(红)。Claude 重点看 thinking signature 是否 100 分;OpenAI 看 usage 字段有没有 Claude / Gemini 残留。

不同协议的 100 分含义不同

Claude 的 100 分包含加密级 thinking signature 验证(Anthropic 服务端签名,中转站不可伪造);OpenAI / Gemini 因为没有同等级的服务端签名机制,验证强度只到行为级 / 协议级。每份报告页顶部都会明确标注本次检测属于哪一级,避免把协议兼容性误读成加密级真伪证明。完整说明见 常见问题。
